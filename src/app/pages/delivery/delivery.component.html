Timeline & deliverables

Phased plan, deliverables, and success metrics from the proposal.

Proposed timeline (MVP ~6 months)

  • Phase 1 – Discovery & Design (3w): Requirements, UI mockups.
  • Phase 2 – Data Foundation (4w): Snowflake schemas, ingestion setup.
  • Phase 3 – API & Duplicate Engine (5w): Core logic and endpoints.
  • Phase 4 – Frontend MVP (4w): Vendor/AP dashboards, filters, exports.
  • Phase 5 – Security Setup (2w): SSO, masking, role-based access.
  • Phase 6 – Pilot & QA (3w): Testing, feedback, refinements.
  • Phase 7 – Launch & Training (2w): Rollout, docs, training.
  • Phase 8 – Continuous Improvement: Enhancements, predictive features.

Expected deliverables

  • Unified VM/AP schema in Snowflake (CUR_VM, CUR_AP, DUP tables).
  • Web portal with dashboards, filters, export/import options.
  • Integrated duplicate detection and history lookup.
  • Role-based security with full audit logs.
  • Documentation and user training materials.

Success metrics

  • Audit turnaround time: 60% faster than current Excel process.
  • Duplicate detection accuracy: ≥95% precision.
  • User adoption rate: ≥90% within 1 month.
  • Query response time: <2 seconds (95th percentile).
  • Security breaches: 0.
